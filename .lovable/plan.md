## Goal
Make the site discoverable on Google by adding SEO essentials and submitting it to Google Search Console.

## What I'll do

### 1. SEO basics in the project
- **`index.html`** — set a real `<title>`, `<meta name="description">`, `<link rel="canonical">`, Open Graph tags (`og:title`, `og:description`, `og:url`, `og:type`), Twitter card tags, and a basic `Organization` / `WebSite` JSON-LD block. Replace any default "Lovable" placeholders.
- **`public/robots.txt`** — allow all crawlers + point to the sitemap.
- **`public/sitemap.xml`** — generated by a small `scripts/generate-sitemap.ts` wired into `predev` / `prebuild`, so it stays in sync. Will include only public, indexable routes (login, public landing, `/inscricoes`, etc.) — NOT the per-role internal app routes (student, professor, coordenador, decano, reitor, finanças, académica, gap), which are auth-gated and shouldn't be indexed.

### 2. Google Search Console verification
- Request a `META` verification token from Google.
- Inject the `<meta name="google-site-verification" ...>` into `index.html`.
- Ask you to **publish** so the tag is live at `https://upra-kortex-teste.lovable.app/`.
- Call Google's verify endpoint, then add the site to your Search Console property list.
- After that, you (or I via the connector) can submit the sitemap inside GSC for faster indexing.

### 3. Important caveats — read these
- **Lovable subdomains rank poorly.** `*.lovable.app` is shared infrastructure; Google rarely ranks these well even when fully optimized. For real search visibility you'll want a **custom domain** (e.g. `upra.ao`). I can guide you through connecting one after publishing.
- **Indexing takes time.** Even once verified and submitted, Google usually takes a few days to a few weeks to show your pages in results.
- **Most of your app is auth-gated.** Only public pages (landing, login, `/inscricoes`, `/candidatar`) can realistically be indexed — Google can't log in to see the dashboards.
- **SPA limitation.** All routes share `index.html`'s metadata. Per-route titles/descriptions need `react-helmet-async`. I'll leave that out for now and only add it if you want richer per-page SEO later.
